# Описание моделей и хаотических модификаций

Файл содержит краткое описание реализованных архитектур, их хаотических модификаций и ограничений на изменения.

---

## 1. ChaoticLogisticNet (резервуар на логистическом отображении)

**Суть модели:**
- Рекуррентная модель, где состояние `h` обновляется через логистическое отображение:
  $$ h_{t+1} = (1 - \beta) h_t + \beta \cdot r_t \cdot h_t (1 - h_t) $$
- Параметр `r_t` зависит от входа через линейный слой.
- Используется как reservoir: обучается только выходной слой.

**Хаотическая модификация:**
- Используется логистическая динамика (классическая хаотическая система).
- `r_t` ограничен диапазоном `[r_min, r_max]`, что задаёт режим (от стабильного к хаотическому).
- Демпфирование (`beta`) делает динамику управляемой.

**Почему хаотическая:**
- Логистическое отображение при `r ≈ 3.5+` даёт чувствительность к начальным условиям.
- Вход влияет на `r_t`, что делает динамику вход-управляемым хаосом.

**НЕЛЬЗЯ ЛОМАТЬ:**
- Ограничение `r_t` через sigmoid + scaling.
- Clamp состояния `h` в `(0,1)` — иначе взрывы или деградация.
- Формулу обновления (особенно структуру `h*(1-h)`).
- Логику `train_r=False` (заморозка reservoir).

## 2. ESN (Echo State Network)

**Суть модели:**
- Классический reservoir computing:
  - фиксированный рекуррентный слой (`W`)
  - обучается только линейный readout (ridge regression)
- Обновление:
  $$ x = (1-\alpha)x + \alpha \tanh(W_{in} u + W x) $$

**Хаотическая модификация отсутствует (базовая модель).**

**НЕЛЬЗЯ ЛОМАТЬ:**
- Масштабирование `W` по spectral radius.
- Разделение: reservoir фиксирован, обучается только `W_out`.
- Ridge-регрессию (регуляризация обязательна для устойчивости).

---

## 3. ChaoticESN

**Суть модели:**
- Тот же ESN, но:
  - spectral radius > 1

**Хаотическая модификация:**
- Увеличенный spectral radius (например, 1.5) переводит систему:
  - из устойчивого режима → в хаотический/критический

**Почему хаотическая:**
- При `ρ(W) > 1` нарушается классическое echo state условие.
- Возникает чувствительность к входам и начальному состоянию.

**НЕЛЬЗЯ ЛОМАТЬ:**
- Spectral radius > 1 (это ключевая модификация).
- Нормализацию матрицы `W`.
- Структуру reservoir (не обучать его градиентом).

## 4. TransientChaoticESN

**Суть модели:**
- ESN с временно затухающим хаосом.
- Используется коэффициент усиления `g_t`:
  $$ preact = W_{in} u_t + g_t \cdot (W x) $$

**Хаотическая модификация:**
- `g_t` экспоненциально уменьшается:
  - сначала хаос (`g_start > 1`)
  - потом стабилизация (`g_end < 1`)

**Почему хаотическая:**
- В начале система в хаотическом режиме.
- Затем переходит в стабильный → сочетание памяти и устойчивости.

**НЕЛЬЗЯ ЛОМАТЬ:**
- Форму gain schedule (экспоненциальное затухание).
- Начальное условие `g_start > 1`.
- Конечное `g_end < 1`.
- Умножение именно на рекуррентную часть (`W @ x`).

## 5. LSTMForecast

**Суть модели:**
- Обычный LSTM (2 слоя) + линейная голова.
- Использует стандартную динамику памяти LSTM.

**Хаотическая модификация отсутствует.**

**НЕЛЬЗЯ ЛОМАТЬ:**
- Структуру LSTM (PyTorch реализация).
- Использование последнего hidden state (`h_n[-1]`).

---

## 6. LSTMForecastChaotic

**Суть модели:**
- Та же LSTM, но с хаотической инициализацией весов.

**Хаотическая модификация:**
- Веса инициализируются через логистическое отображение:
  $$ x_{n+1} = r x_n (1 - x_n) $$

**Почему хаотическая:**
- Последовательность значений обладает:
  - псевдослучайностью
  - чувствительностью к параметру `r`
- Это создаёт более сложное начальное распределение весов.

**НЕЛЬЗЯ ЛОМАТЬ:**
- Функцию `init_chaotic_`.
- Использование logistic map (не заменять на random).
- Масштабирование (аналог Xavier).
- Обнуление bias.

## 7. VanillaMLP

**Суть модели:**
- Обычная MLP:
  - Linear → ReLU → Linear

**Хаотическая модификация отсутствует.**

---

## 8. ChaoticMLP

**Суть модели:**
- MLP, но с хаотической активацией.

**Хаотическая модификация:**
- Вместо ReLU используется:
  - tanh → logistic map → rescale

**Почему хаотическая:**
- Нелинейность включает логистическое отображение.
- Это добавляет:
  - чувствительность к входу
  - сложную форму отклика

**НЕЛЬЗЯ ЛОМАТЬ:**
- Последовательность:
  `tanh → [0,1] → logistic → [-1,1]`
- Параметр `r` (контролирует режим хаоса).
- Не заменять на стандартные активации.

# Общие принципы (важно)

1. **Хаос задаётся строго через:**
   - logistic map
   - spectral radius
   - временные коэффициенты (g_t)

2. **Нельзя:**
   - заменять хаотические компоненты на random/шум
   - убирать ограничения диапазонов
   - менять scaling (это ломает динамику)

3. **Можно:**
   - менять размерности (hidden_size, n_reservoir)
   - менять readout
   - добавлять регуляризацию на выходе

4. **Reservoir-подход:**
   - внутреннюю динамику не обучаем (или строго контролируем)
   - обучается только выходной слой (если явно не указано иначе)
